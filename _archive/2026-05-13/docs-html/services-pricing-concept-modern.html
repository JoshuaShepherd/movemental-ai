The model

How the model works.

Movemental’s work falls into two layers.

Layer one is foundational: Safety, Sandbox, and Skills.

Layer two is applied: Solutions.

The first layer is where organizations reduce risk, gain clarity, and build capability. The second is where systems, workflows, and deployments become viable.

Layer one

Foundational services.

These are the first three stages of the path. They are high-touch, guided, and relational by design.

  1. 01 · Safety

    Safety

    Clarity, governance, and boundaries

    We help organizations define the principles, policies, theological positions, and data boundaries required to move wisely.

    Indicative Starting at $15,000
  2. 02 · Sandbox

    Sandbox

    Guided discovery and bounded experimentation

    We help organizations explore real use cases, test tools safely, and learn in structured environments before they commit to deployment.

    Indicative Starting at $10,000
  3. 03 · Skills

    Skills

    Formation, training, and internal capability

    We help teams develop the discernment, fluency, and leadership capacity needed to use AI well in real organizational settings.

    Indicative Starting at $12,000

These engagements can stand alone, but they are strongest when sequenced together.

Why the sequence

Why the sequence changes pricing.

The old model assumes systems can be sold first and adopted later. The new model recognizes that systems only work when the organization is ready for them.

That changes both value and pricing.

The first three stages are where confusion is reduced, trust is built, and readiness is formed. The fourth is where that readiness is expressed through durable systems.

Common questions

Common questions.

  • Can we start with a solution build?

    Sometimes — but only if the foundational work already exists internally. If not, starting with deployment usually produces shallow adoption and unnecessary risk.

  • Do we have to do all three foundational stages with you?

    Not always. But the order still matters. If the work is already present, we can build from there. If it is missing, we help create it.

  • Are solutions products or services?

    They are applied deployments. Some are repeatable patterns. Some are custom. All of them depend on the foundation.
