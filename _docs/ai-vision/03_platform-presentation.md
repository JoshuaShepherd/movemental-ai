# AI Presentation Guide: Movemental.ai Platform

> **How to Frame AI on the Main Movemental.ai Platform**

**Version**: 1.0.0  
**Last Updated**: January 2026  
**Purpose**: Specific guidance for presenting AI capabilities on Movemental.ai

---

## Core Principle

**Movemental.ai is about the NETWORK, not individual leaders.**

AI presentation should emphasize scenius, collective intelligence, network effects, and the platform as a destination for transformation—not individual leader tools or personal voice preservation.

---

- Ignore network connections

- **Why:** This is about the network, not individual features
- **Exception:** Only mention if highlighting how network benefits individual leaders

- **Why:** That's individual site messaging, not platform messaging
- **Exception:** Only if explaining how network preserves individual voices

### Individual Analytics
- **Why:** Platform is about collective, not individual metrics
- **Exception:** Only if showing how network amplifies individual success

### Generic AI Capabilities
- **Why:** Emphasize movemental DNA and scenius, not generic features
- **Exception:** Only if explaining how movemental DNA enhances generic capabilities

## Common Pitfalls to Avoid

### 1. Making It Sound Individual-Focused
- ❌ "AI tools for movemental leaders"
- ✅ "Scenius-enhanced network intelligence"

### 2. Emphasizing Personal Over Network
- ❌ "Your personal AI assistant"
- ✅ "Collective intelligence serving the network"

### 3. Focusing on Tools Over Destination
- ❌ "Platform with AI features"
- ✅ "Destination for movemental thinking"

### 4. Making It Sound Generic
- ❌ "AI-powered content platform"
- ✅ "Movemental Intelligence network"

### 5. Ignoring Scenius
- ❌ "Individual leaders with AI tools"
- ✅ "Scenius-enhanced network where ideas multiply"
