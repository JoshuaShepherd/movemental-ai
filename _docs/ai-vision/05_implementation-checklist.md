## Quality Checklist (Before Moving to Next Phase)

For each completed component, verify:

- [ ] Content is complete and reviewed
- [ ] UI mockup matches Mobbin reference quality
- [ ] Mobile responsive design included
- [ ] Accessibility considered (WCAG 2.1 AA)
- [ ] Integration with other components tested
- [ ] User flow makes sense
- [ ] CTAs are clear and prominent
- [ ] Loading states considered
- [ ] Error states considered
- [ ] Empty states considered

---

## Notes

- **Start with content** - Write content before finalizing UI
- **Reference liberally** - Use Mobbin to find patterns, not copy exactly
- **Test flows** - Ensure user journeys make sense
- **Mobile-first** - Design mobile, then enhance for desktop
- **Iterate** - Don't perfect one page before moving to next
- **Document decisions** - Note why you chose certain patterns
