# Movemental Gate & Pipeline Page (`/gate`)

## Purpose
Explain how Movemental evaluates and onboards leaders/organizations through the Gate rubric, showcases active pipeline initiatives, and enables trusted referrals.

## Audience
- Existing network members referring peers.
- Prospective leaders wanting transparency on selection.
- Partners validating diligence.

## Outcomes
1. Visitors understand the 100-point rubric and readiness tiers.
2. Pipeline dashboard communicates current mix (High/Medium/Emerging) and action plans.
3. Secure referral CTA captures qualified leads.

## Component Map
1. **Intro Banner**
   - Statement: “Movemental is invitation-only, governed by a 100-point Gate rubric.”
2. **Rubric Overview**
   - Table displaying pillars (Movement Alignment, Content Velocity, Audience, Economics, Platform Mindset, Network Contribution, Governance Fit) with point ranges and indicators.
3. **Archetype Profiles**
   - Cards for Alan Hirsch (Platform Architect) + Brad Brisco (Network Amplifier) explaining evaluation lens.
4. **Readiness Bands**
   - Explanation of High (85+), Medium (70–84), Emerging (55–69), Monitor (<55) with expectations and next steps.
5. **Pipeline Dashboard**
   - Metrics: counts per readiness tier + entity type (Individual, Org, Research).
   - Action tags for notable initiatives (Dave launch, Forge America deployment, Soul of Movement microsite, Send Institute research hub, Forge Canada localization, Geoff Maddock content sprint).
6. **Case Spotlights**
   - Short blurbs linking to case studies demonstrating each readiness tier in action.
7. **Referral CTA**
   - Summary of submission requirements + button to secure referral form.
8. **FAQ**
   - Answers about timing, confidentiality, selection committee, follow-up process.

## Data Needs
- Rubric copy from Gate document.
- Live pipeline metrics from CRM.
- Case study summaries + links.
- Referral form endpoint.

## Tone
- Transparent, rigorous, warmly invitational to the right leaders.
- Reinforce Movemental’s stewardship mindset (“We treat every platform as shared infrastructure.”)
