# Pipeline Admin (Internal Staff Module)

## Purpose
Equip Movemental staff with tools to manage leader/org/research pipelines, evaluate candidates via the Gate rubric, coordinate onboarding tasks, and report readiness stats to stakeholders.

## Key Capabilities
1. View/manage full pipeline (Engage, Prototype, Nurture, Monitor).
2. Score candidates against rubric with evidence attachments.
3. Trigger actions (send discovery invite, schedule design lab, archive candidate).
4. Generate reports for leadership/investors.

## Component Map
1. **Pipeline Overview**
   - Kanban or table view segmented by readiness tier.
   - Filters by type (Individual, Org, Research), geography, referring partner.
2. **Candidate Detail Drawer**
   - Profile info, dossier links, rubric scores, notes, attachments.
   - Action buttons (schedule call, send resources, move stage).
3. **Scoring Interface**
   - Form for updating pillar scores with evidence references.
   - History log of score changes.
4. **Task & Reminder Panel**
   - Follow-up tasks, due dates, owner assignments.
5. **Reporting Dashboard**
   - Metrics on pipeline volume, conversion, time-in-stage.
   - Export/print options for leadership decks.
6. **Referral Intake Queue**
   - New submissions awaiting triage, ability to assign reviewer.

## Data Requirements
- Candidate database (from research dossiers).
- Task/CRM system integration.
- Audit log for changes (compliance).

## UX Considerations
- Role-based access (staff vs. advisors).
- Bulk update tools for cohort processing.
- Quick links to share updates with Alan/Brad/partners.
