## Prompt Structure

Each pillar article prompt follows this structure:

1. **Article Metadata** - Portal theme, primary keyword, word count, SEO targets
2. **Source Material References** - Primary sources (`_docs/extracted-books/`), supporting sources, related content
3. **Content Requirements** - Complete article structure (7 sections: Introduction, Framework Foundation, Practical Application, Case Studies, Common Challenges, Integration, Conclusion)
4. **Voice Fidelity Requirements** - Essential voice elements, voice consistency checklist
5. **SEO Specifications** - Primary/secondary keyword targeting, internal linking strategy, meta description
6. **Quality Checklist** - Content completeness, voice fidelity, SEO optimization, source material integration
7. **Execution Instructions** - Step-by-step process for using the prompt

---

## Usage Instructions

### Step 1: Select Portal and Primary Keyword

1. Navigate to portal directory: `_docs/content/articles/{portal-slug}/`
2. Select pillar article prompt: `pillar-{keyword-slug}.md`

### Step 2: Review Prompt Requirements

1. Read complete prompt specifications
2. Review source material references
3. Understand voice fidelity requirements
4. Note SEO specifications

### Step 3: Reference Source Materials

1. Read primary sources from `_docs/extracted-books/`
2. Review supporting sources from `_docs/content/books/`
3. Study voice documentation from `_docs/04-features/voice/`

### Step 4: Create Article

1. Follow article structure exactly (7 sections)
2. Integrate source material concepts and examples
3. Apply voice fidelity requirements throughout
4. Include required historical examples
5. Use movement/DNA metaphors consistently

### Step 5: Verify Quality

1. Check content completeness (all sections, 3,000+ words)
2. Verify voice fidelity (checklist completion)
3. Optimize SEO (keyword integration, internal links)
4. Verify source material integration
