## How to Use

1. **Find Reference Images**: Use Mobbin or other design inspiration sites
2. **Save to Appropriate Directory**: Place screenshots in the matching feature directory
3. **Name Files Clearly**: Use descriptive names like `mobbin-progressive-form-1.png`
4. **Document Sources**: Consider adding a `sources.md` file in each directory listing where images came from

---

## Reference Image Guidelines

### What to Look For

- **UI Patterns**: Layouts, component arrangements, visual hierarchy
- **User Flows**: Multi-step processes, navigation patterns
- **Mobile Designs**: Responsive layouts, mobile-first patterns
- **Interaction Patterns**: Hover states, transitions, micro-interactions

### What NOT to Include

- ❌ Exact copies of competitor designs (use for inspiration only)
- ❌ Copyrighted material without permission
- ❌ Low-quality screenshots
- ❌ Outdated design patterns

### File Naming Convention

Use descriptive names:
- `mobbin-[pattern-name]-[number].png`
- `reference-[source]-[description].png`
- `inspiration-[feature]-[variant].png`

Example:
- `mobbin-progressive-form-1.png`
- `reference-stripe-checkout-progress.png`
- `inspiration-timeline-vertical.png`
