## 2. Through-line for cohesion (do not break without updating outline)

When editing any chapter, preserve these **global** threads:

1. **Credibility crisis is real** but the book refuses panic and refuses false certainty.
2. **AI as both problem and solution** — never resolve this into a tidy verdict; formation is in holding the tension.
3. **Amplification not replacement** — the central operating distinction; later chapters should **reference** it rather than re-prove it from scratch.
4. **Voice + scenius + transparency (+ optional architecture)** — individual voice (Ch 5–8), communal verification (Ch 9), public accounting (Ch 10), bounded-scale infrastructure sketch where it serves the argument (Ch 10A).
5. **Embodied / relational / formation** goods are **protected** (Ch 13–14); practice chapters (15–18) show how to **use** tools without reversing those protections.
6. **No product pitch** — Movemental may illustrate; it must not become the book’s thesis.

## 3. Cross-chapter rules (agents and editors)

1. **Read the outline slice first** — For chapter *N*, read the matching section in `BOOK_OUTLINE.md` so you know merge history (what used to be separate chapters elsewhere).
2. **Check neighbors** — Open the chapters **before and after in §1a order** (not filename sort). For the preface (`00`), neighbors are **Ch 3** and **Ch 4**; fix dangling “last chapter we…” references if merges renamed topics.
3. **Deduplicate** — If the same statistic or anecdote appears in two chapters, keep it where it **does the most work**; elsewhere replace with a short cross-reference (“As in chapter one…”).
4. **Statistics** — Follow the verified-claims table in [`../CANONICAL_BOOK_DRAFTING_SYSTEM.md`](../CANONICAL_BOOK_DRAFTING_SYSTEM.md) Section 2; do not introduce new numbers without a named primary source.
5. **Voice** — Run changed passages against `BOOK_VOICE_AND_STYLE_GUIDE.md` (anti-slop list, prophetic-pastoral balance).
6. **Meta constraints** — `BOOK_META_OVERVIEW.md` §2: avoid how-to manual drift, manifesto drift, and hype; this is a formation book.

**Deepening a single chapter** — Use [`../../build/prompts/book-chapter-ground-truth-deepening.md`](../../build/prompts/book-chapter-ground-truth-deepening.md) with the chapter path; it already encodes dedup and voice rules.

**Publisher-quality line and structure pass** — When outline and draft exist but the chapter needs **developmental + substantive line + sensitive copy** in one run (memo + edited markdown), use [`../../build/prompts/book-chapter-publisher-quality-edit.md`](../../build/prompts/book-chapter-publisher-quality-edit.md).

## 5. Folder hygiene and legacy material

- **`manuscript-final/`** — Older chapter numbering and splits (standalone SEO, theological integrity, separate agent chapters, etc.). **Quarry only:** when lifting prose, **reconcile** to the merge decisions in `BOOK_OUTLINE.md` (e.g. theological integrity belongs inside Ch 13, not a standalone file).
- **`book-draft/`** — Exploratory drafts; same rule: align to outline before treating text as canonical.

## 6. Suggested agent checklist (before submitting a chapter edit)

- [ ] Outline section for this chapter re-read; no drift into cut topics (e.g. standalone SEO manifesto).
- [ ] Opening and closing lines mesh with adjacent files.
- [ ] No duplicate long blocks from neighbor chapters.
- [ ] Claims and stats traceable or softened per canonical epistemic rules.
- [ ] Voice guide pass on the diff.
- [ ] Meta-overview: still serving movement leaders who steward content (not a generic tech book).
