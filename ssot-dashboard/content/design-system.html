Design System & Tokens

Typography, color tokens, and component patterns for the dashboard and platform.

Dashboard Tokens (this SSOT)

Defined in css/tokens.css: semantic colors (--ssot-bg, --ssot-fg, --ssot-accent, --ssot-sidebar-*), type scale (--ssot-text-xs through --ssot-text-3xl), spacing (--ssot-space-1 through --ssot-space-12), sidebar width. Use tokens only—no hardcoded colors or sizes in components.

Platform Design (React app)

Tailwind + shadcn: Design tokens in app/globals.css and tailwind.config.ts. Components in components/ui/ are shadcn-managed—do not modify directly. Use semantic color names (accent, primary, secondary, muted). Typography: see _docs/_guides/typography-2026.md for font pairings (Inter, Playfair Display, etc.) and variant assignments.

UI Principles (from UI/UX proposal)

Progressive disclosure, visual hierarchy, reading-optimized layout, card-based patterns, step indicators, clean typography. Mobile-first; trust through transparency.
